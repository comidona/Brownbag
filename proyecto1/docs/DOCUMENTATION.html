SCREEN-01

🏠 Pantalla de Inicio

Landing page principal con hero animado, estadísticas del universo Rick & Morty y personajes destacados. Actúa como hub central de navegación.

Elementos UI:
  • Hero section con emoticono 🛸 flotante (animación float)
  • Título con gradiente verde-cyan
  • 3 tarjetas de estadísticas (personajes, episodios, ubicaciones)
  • Grid de personajes destacados (6 primeros)
  • Sección CTA con call-to-action
Funcionalidades:
  • Carga automática de stats desde la API (count de info)
  • Skeleton loading en tarjetas de personajes
  • Navegación rápida a todas las secciones
  • Animaciones de hover en cards
GET /character GET /episode GET /location → Personajes → Episodios → Ubicaciones
SCREEN-02

👽 Personajes

Explorador de personajes con vista en grid, filtros avanzados y paginación. Muestra tarjetas con imagen, nombre, especie y estado vital con indicadores de color.

Elementos UI:
  • Barra de búsqueda por nombre
  • 4 filtros: Estado (Alive/Dead/unknown), Especie, Género, Búsqueda
  • Grid responsivo de tarjetas de personaje
  • Badge de estado con indicador de color (verde=vivo, rojo=muerto, gris=desconocido)
  • Controles de paginación (Anterior/Siguiente) con indicador de página
  • Skeleton loading (8 placeholders)
  • Estado vacío con emoji 🔍
Funcionalidades:
  • Filtrado combinado por nombre + status + species + gender
  • Paginación con 20 resultados por página
  • Click en tarjeta navega al detalle
  • Debounce implícito en búsqueda (re-render)
GET /character?page=&name=&status=&gender= → Detalle de Personaje
SCREEN-03

📋 Detalle de Personaje

Vista completa de un personaje individual con toda su información, lista de episodios en los que aparece y funcionalidad de favoritos.

Elementos UI:
  • Botón "← Volver" a la lista
  • Imagen grande del personaje con borde redondeado
  • Nombre, especie, género, estado (con badge de color)
  • Información de origen y última ubicación conocida
  • Botón de favorito (⭐ toggle) con efecto hover
  • Lista de episodios con código de episodio (S01E01) y nombre
  • Skeleton loading mientras carga
Funcionalidades:
  • Carga datos del personaje por ID
  • Carga múltiples episodios en una sola petición
  • Toggle de favorito persistido en localStorage (key: 'rickpedia-favs')
  • Extracción de IDs de episodio desde URLs
GET /character/:id GET /episode/[ids] → Personajes (volver)
SCREEN-04

📺 Episodios

Explorador de episodios con filtro por temporada y tarjetas con código de episodio, nombre y fecha de emisión.

Elementos UI:
  • Botones de filtro por temporada (S01-S05 + Todas)
  • Tarjetas de episodio con código de temporada (badge verde)
  • Nombre del episodio, fecha de emisión
  • Contador de personajes por episodio
  • Paginación
  • Skeleton loading
Funcionalidades:
  • Filtrado client-side por temporada (parsea código S01E01)
  • Paginación con 20 episodios por página
GET /episode?page=
SCREEN-05

🌍 Ubicaciones

Explorador de ubicaciones del multiverso con tarjetas que muestran tipo, dimensión y cantidad de residentes.

Elementos UI:
  • Grid de tarjetas de ubicación
  • Nombre con icono dinámico según tipo
  • Tipo de ubicación (Planet, Space station, etc.)
  • Dimensión con color diferenciado
  • Contador de residentes
  • Paginación
Funcionalidades:
  • Paginación API con 20 ubicaciones por página
  • Colores de dimensión diferenciados (C-137, desconocida, etc.)
GET /location?page=
SCREEN-06

⭐ Favoritos

Lista de personajes marcados como favoritos por el usuario. Los favoritos se almacenan en localStorage y persisten entre sesiones.

Elementos UI:
  • Grid de tarjetas de personajes favoritos
  • Botón de eliminar favorito (❌) por tarjeta
  • Estado vacío con mensaje motivacional y emoji ⭐
  • Link a personajes si la lista está vacía
Funcionalidades:
  • Lectura de IDs favoritos desde localStorage (key: 'rickpedia-favs')
  • Carga batch de personajes por IDs múltiples
  • Eliminación individual de favoritos con actualización inmediata
  • Navegación al detalle desde cada tarjeta
GET /character/[ids] → Detalle de Personaje → Personajes

🔌 API & Servicios

API REST pública sin autenticación.

Base URL: https://rickandmortyapi.com/api

Nota: Todos los endpoints de lista devuelven respuestas paginadas con un objeto info que contiene count, pages, next y prev. Cada página contiene 20 resultados.
GET /character

Obtiene una lista paginada de personajes con filtros opcionales.

Parámetros

ParámetroTipoRequeridoDescripción
pagenumberNoNúmero de página (default: 1)
namestringNoFiltrar por nombre del personaje
statusstringNoFiltrar por estado: alive, dead, unknown
speciesstringNoFiltrar por especie
genderstringNoFiltrar por género: female, male, genderless, unknown

Respuesta de Ejemplo

{
  "info": {
    "count": 826,
    "pages": 42,
    "next": "https://rickandmortyapi.com/api/character?page=2",
    "prev": null
  },
  "results": [
    {
      "id": 1,
      "name": "Rick Sanchez",
      "status": "Alive",
      "species": "Human",
      "type": "",
      "gender": "Male",
      "origin": { "name": "Earth (C-137)", "url": "..." },
      "location": { "name": "Citadel of Ricks", "url": "..." },
      "image": "https://rickandmortyapi.com/api/character/avatar/1.jpeg",
      "episode": ["https://rickandmortyapi.com/api/episode/1", "..."],
      "url": "https://rickandmortyapi.com/api/character/1",
      "created": "2017-11-04T18:48:46.250Z"
    }
  ]
}
GET /character/:id

Obtiene información detallada de un personaje específico. También soporta múltiples IDs separados por coma.

Parámetros

ParámetroTipoRequeridoDescripción
idnumber | number[]SíID del personaje o lista de IDs separados por coma

Respuesta de Ejemplo

{
  "id": 1,
  "name": "Rick Sanchez",
  "status": "Alive",
  "species": "Human",
  "type": "",
  "gender": "Male",
  "origin": { "name": "Earth (C-137)", "url": "https://rickandmortyapi.com/api/location/1" },
  "location": { "name": "Citadel of Ricks", "url": "https://rickandmortyapi.com/api/location/3" },
  "image": "https://rickandmortyapi.com/api/character/avatar/1.jpeg",
  "episode": [
    "https://rickandmortyapi.com/api/episode/1",
    "https://rickandmortyapi.com/api/episode/2"
  ],
  "url": "https://rickandmortyapi.com/api/character/1",
  "created": "2017-11-04T18:48:46.250Z"
}
Tip: Para obtener múltiples personajes (ej: favoritos), usar /character/1,2,3. Si es un solo ID devuelve un objeto; si son múltiples, devuelve un array.
GET /episode

Obtiene una lista paginada de episodios de la serie.

Parámetros

ParámetroTipoRequeridoDescripción
pagenumberNoNúmero de página (default: 1)
namestringNoFiltrar por nombre del episodio
episodestringNoFiltrar por código de episodio (ej: S01E01)

Respuesta de Ejemplo

  "results": [

      "name": "Pilot",

      "created": "2017-11-10T12:56:33.798Z"
    }
  ]
}
GET /episode/:id

Obtiene detalle de un episodio específico por su ID.

Parámetros

ParámetroTipoRequeridoDescripción
idnumberSíID del episodio

Respuesta de Ejemplo

{
  "id": 1,
  "name": "Pilot",
  "air_date": "December 2, 2013",
  "episode": "S01E01",
  "characters": ["https://rickandmortyapi.com/api/character/1", "..."],
  "url": "https://rickandmortyapi.com/api/episode/1",
  "created": "2017-11-10T12:56:33.798Z"
}
GET /episode/1,2,3

Obtiene múltiples episodios en una sola petición. Se usa en la pantalla de detalle del personaje para cargar todos sus episodios.

Parámetros

ParámetroTipoRequeridoDescripción
idsnumber[]SíLista de IDs separados por coma en la URL
Importante: Si se pasa un solo ID devuelve un objeto Episode. Si se pasan múltiples IDs devuelve un array Episode[]. El código de la app normaliza ambos casos con Array.isArray().

  ]
}
